## 需求概述

全面完善照片集管理后台的所有缺失功能模块，按优先级逐步实现。

## 核心功能

### 1. 会员套餐管理（最高优先级）

后端提供管理员级别的会员套餐 CRUD 接口（创建、编辑、上架/下架、删除），前端新增会员管理页面，支持套餐列表展示、新增/编辑套餐表单、上下架状态切换。

### 2. 重启后端 API

后端实现 `/admin/system/restart` 接口，前端 SiteSettings.vue 已调用该接口但后端未实现。

### 3. 管理员直接编辑套图

管理员可以跳过审核流程，直接编辑任意套图的内容（标题、封面、描述、标签、价格、图片等）。

### 4. 批量操作

支持批量审核通过、批量审核拒绝、批量删除套图，提升管理效率。

### 5. 数据导出

支持用户列表导出和订单列表导出为 CSV 格式。

### 6. 用户详情增强

用户详情增加收藏数量统计，增强信息展示。

### 7. 已知 Bug 修复

- 修复 `user_repository.go` 中 status 筛选逻辑错误
- 修复 `OrderManage.vue` 前端传参 `limit` 与后端期望 `size` 不匹配的问题

## 技术栈

- 后端：Go + Gin + GORM + Redis
- 前端：Vue 3 + Element Plus + ECharts
- 数据库：MySQL
- 认证：JWT（需 admin 角色）

## 实现方案

### 整体策略

遵循现有的 Repository → Service → Handler 三层架构，在现有代码基础上扩展。后端新增路由挂载在 `/api/admin` 分组下，前端新增页面遵循现有的 Vue 3 + Element Plus 组件模式。

### 各模块实现方案

**1. 会员套餐管理**

- 扩展 `MembershipRepository`：新增 Create、Update、Delete、ListAll（含分页）方法
- 扩展 `MembershipHandler`：新增 AdminList、AdminCreate、AdminUpdate、AdminDelete 方法
- 新增路由：`/admin/memberships` 的 CRUD 路由
- 前端：新增 `MembershipManage.vue` 页面，复用 Element Plus 表格+表单模式

**2. 重启后端 API**

- 在 admin_handler.go 新增 `RestartServer` 方法
- 使用 `os.Exit(0)` 配合外部进程管理器（Docker restart policy 或 supervisord）实现优雅重启
- 新增路由：`POST /admin/system/restart`

**3. 管理员编辑套图**

- 后端已有 `photosetHandler.Update` 支持 admin 角色，只需确保前端路由正确
- 前端在 ContentReview 页面增加"编辑"按钮，跳转到 EditPhotoset 页面
- 确保 admin 角色可以编辑任意套图（已有权限校验）

**4. 批量操作**

- 后端新增批量接口：`POST /admin/photosets/batch/approve`、`POST /admin/photosets/batch/reject`、`DELETE /admin/photosets/batch`
- 前端在 ContentReview 页面增加批量选择复选框和批量操作按钮

**5. 数据导出**

- 后端新增导出接口：`GET /admin/users/export`、`GET /admin/orders/export`
- 返回 CSV 格式的文件流
- 前端增加导出按钮

**6. 用户详情增强**

- 扩展 `FindByIDWithStats` 方法，增加收藏数查询
- 后端已有基础设施，只需增加 Favorite 表的 Count 查询

**7. Bug 修复**

- `user_repository.go` 第78行：修复 status 筛选条件
- `OrderManage.vue`：修正传参字段名 `limit` → `size`

## 性能与可靠性

- 批量操作使用事务确保原子性
- 导出接口使用流式输出避免大文件 OOM
- 重启接口需二次确认，防止误操作
