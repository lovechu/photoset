# 增量 PRD：社区分类管理

## 1. 项目信息

| 项 | 值 |
|---|---|
| 语言 | 中文 |
| 技术栈 | Go + Gin（后端），Vue 3 + Element Plus（管理后台） |
| 项目名 | `photoset_community_category` |
| 原始需求 | 管理员可在后台对帖子分类进行新增、编辑、删除、排序，管理后台新增"分类管理"菜单项 |

## 2. 产品定义

### 2.1 产品目标

1. **管理员可完整管理分类**：支持分类的增删改查与排序，满足社区运营灵活调整分类体系的需求。
2. **数据完整性保护**：删除分类时校验关联帖子，有帖子的分类不可删除，防止数据断联。
3. **与现有管理后台无缝融合**：交互风格、路由结构、API 设计均延续现有 5 个社区管理页面的模式。

### 2.2 用户故事

| # | 角色 | 场景 | 价值 |
|---|---|---|---|
| US1 | 管理员 | 在后台查看所有帖子分类列表 | 了解当前分类结构与每个分类的使用情况 |
| US2 | 管理员 | 创建新分类，填写名称、标识 key、排序号、颜色 | 根据社区发展新增分类，灵活扩展分类体系 |
| US3 | 管理员 | 编辑已有分类的属性（名称/颜色/排序等） | 修正错误或优化分类信息 |
| US4 | 管理员 | 删除不再需要的分类 | 保持分类列表整洁，降低用户选择成本 |
| US5 | 管理员 | 拖拽调整分类排序 | 控制前端分类展示顺序，突出核心分类 |

## 3. 技术规范

### 3.1 需求池

| 优先级 | 需求 | 说明 |
|---|---|---|
| **P0** | 后端 CRUD API | `GET/POST/PUT/DELETE /api/admin/community/categories`，含排序更新接口 |
| **P0** | 前端分类管理页面 | Table 展示 + Dialog 新增/编辑 + Popconfirm 删除 |
| **P0** | 管理后台菜单集成 | 左侧菜单新增"分类管理"，路由 `/admin/community/categories` |
| **P1** | 删除保护 | 删除时检查该分类下帖子数，> 0 则禁止删除并提示 |
| **P1** | 排序能力 | 支持上移/下移或拖拽排序，排序号自动重排 |
| **P1** | 颜色字段 | 分类表增加 `color` 字段（hex），前端用 ColorPicker |
| **P2** | 分类帖子计数 | 列表展示每个分类下的帖子数量 |
| **P2** | 图标字段 | 分类表增加 `icon` 字段（可选，为后续前端展示预留） |
| **P2** | 操作日志 | 记录分类变更（谁/何时/做了什么） |

### 3.2 数据模型（增量）

在现有 `categories` 表基础上增加/确认字段：

```
categories:
  id            INT PRIMARY KEY AUTO_INCREMENT
  key           VARCHAR(64)  NOT NULL UNIQUE   -- 标识 (如 discuss)
  name          VARCHAR(128) NOT NULL          -- 显示名称
  description   VARCHAR(256)                   -- 描述（可选）
  color         VARCHAR(7)   DEFAULT '#409EFF' -- hex 颜色
  icon          VARCHAR(64)  DEFAULT ''        -- 图标标识（P2）
  sort_order    INT          DEFAULT 0         -- 排序号
  created_at    DATETIME
  updated_at    DATETIME
```

### 3.3 API 设计

| 方法 | 路径 | 说明 | 优先级 |
|---|---|---|---|
| `GET` | `/api/admin/community/categories` | 分类列表（含帖子计数） | P0 |
| `POST` | `/api/admin/community/categories` | 创建分类 | P0 |
| `PUT` | `/api/admin/community/categories/:id` | 编辑分类 | P0 |
| `DELETE` | `/api/admin/community/categories/:id` | 删除分类（校验帖子数） | P0 |
| `PUT` | `/api/admin/community/categories/sort` | 批量更新排序 | P1 |

> 管理端接口统一加 `/admin` 前缀，与公开 `GET /api/community/categories` 区分。

```
│  ┌─────────────┐               │  ┌─────────────────┐  │
│  │ 社区管理    │               │  │ [+ 新增分类]    │  │
│  │  帖子管理   │               │  │                 │  │
│  │  回帖管理   │               │  │ 名称  Key  排序  │  │
│  │  敏感词     │               │  │ 颜色  帖子数 操作│  │
│  │  举报管理   │               │  │ ─────────────── │  │
│  │  用户积分   │               │  │ 讨论 discuss 1  │  │
│  │▶ 分类管理  │               │  │ 🔵 128  编辑 删除│  │
│  └─────────────┘               │  │ 求助 question 2 │  │
│                                │  │ 🟢 56   编辑 删除│  │
│                                │  │ ...             │  │
│                                │  └─────────────────┘  │
└─────────────────────────────────────────────────────────┘

新增/编辑弹窗：
┌─────────────────────────┐
│  新增分类 / 编辑分类     │
│                         │
│  名称：[____________]   │
│  标识：[____________]   │ (新建时可填，编辑时只读)
│  排序：[___]            │
│  颜色：[🔵 #409EFF  ▾] │ (Element Plus ColorPicker)
│                         │
│       [取消]  [保存]    │
└─────────────────────────┘
```

- **列表页**：Element Plus `<el-table>` + `<el-pagination>`（若分类 > 20 条）
- **新增/编辑**：`<el-dialog>` 内嵌 `<el-form>`，含校验（名称必填、key 唯一且仅允许小写字母+下划线）
- **删除**：`<el-popconfirm>` 确认，若关联帖子则 `<el-alert type="warning">` 提示无法删除
- **排序**：表格行内上移/下移按钮（P1），或直接修改排序号输入框
- **颜色**：`<el-color-picker>` 组件
- **菜单**：在 `AdminLayout.vue` 的社区管理子菜单中增加"分类管理"项，路由 `/admin/community/categories`
